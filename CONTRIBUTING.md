# Contributing

beo has high standards for quality, this isn't because I don't like changes, but
because I want good things.

In general, here are a few easy rules:

- No AI-generated code, no exceptions.
- Avoid pulling in dependencies
- Keep things simple
- Try to be a "perfect" package
  - Document every external API
  - Strive for a small bundle size
  - Strive for a simple API
